GUIA FRONTEND MERCAPLENO (REACT + VITE)
Ruta del proyecto frontend:


============================================================
1) RESPUESTA DIRECTA A TU DUDA
============================================================
Si: el frontend es quien CONSUME la API.
Y el backend es quien EXPONE/ATIENDE esas rutas.

En este proyecto:
- Frontend consume rutas como /api/auth/login, /api/sales/products, etc.
- Backend las atiende en controllers NestJS (auth.controller.ts, sales.controller.ts, etc.).

============================================================
2) FLUJO GENERAL DE LA APP
============================================================
1. El navegador abre la app Vite.
2. src/main.jsx monta app/root.jsx.
3. root.jsx envuelve App dentro de providers.jsx.
4. providers.jsx activa BrowserRouter + AuthProvider.
5. App.jsx define rutas publicas y protegidas por rol.
6. Las paginas hacen requests HTTP con httpRequest() (app/lib/api/httpClient.js).
7. httpRequest usa API_URL definida en app/lib/config/env.js (VITE_API_URL).
8. Backend responde en rutas /api/*.

3.6 app/lib/api/
- app/lib/api/httpClient.js
  Cliente HTTP unificado.
  Bloques:
  - normalizePath(): normaliza ruta.
  - extractErrorMessage(): arma mensaje de error.
  - buildApiUrl(): API_URL + path.
  - httpRequest():
    - agrega Authorization si auth=true.
    - serializa JSON o FormData.
    - soporta responseType blob.
    - parsea respuesta JSON/texto.
    - lanza Error con status y data en respuestas no OK.

7) CORRECCION APLICADA EN ESTA SESION
============================================================
Archivo corregido:
- app/routes/Lista_productos.jsx

Problema detectado:
- El CRUD de productos llamaba /api/productos sin token.
- Ese modulo en backend esta protegido por JWT (guard global).

Cambio aplicado:
- Se agrego envio de autenticacion en GET/POST/PUT/DELETE:
  auth: true,
  token
- Se agrego manejo basico para 401/403 (logout + mensaje).

Resultado:
- El CRUD de productos queda alineado con el backend protegido.

Verificacion tecnica:
- Se ejecuto build del frontend con exito:
  npm run build
